# SDSN_TSC_Indika
Case study resources for TSC paper

Note that some of the terminologies used in the executable (XML) service network design differ from what in the paper and textual DSL. However, the capabilities are the same (just canonical names differ).  We use “regulation unit” for all units of regulations (i.e., regulation designs in collaboration units, inter-collaboration, inter-process, and inter-VSN). We use “behavior unit” for “collaboration unit”. 

New : the updated roadside case study reflects the above changes.
